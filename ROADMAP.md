## v1.0 Stability Guarantee

Before v1.0.0, the following interfaces must be stable and extensible without breaking changes:

- **Checker interface**: `Check(dir, opts) []Result` - must support new languages without signature changes
- **Action interface**: `Run()`, `Propose()`, `Apply()` - must support new action types
- **Result struct**: Adding fields is allowed; removing/renaming is breaking
- **Options struct**: Must use extensible pattern (Extra map) for language-specific options
- **LanguageConfig**: Must use extensible pattern for language-specific configuration
- **CLI commands**: `check`, `validate`, `release`, etc. - names and core flags are stable
- **Configuration file**: `.releaseagent.yaml` format - new keys allowed, existing keys stable

The test for v1.0 readiness: Can we add Python, Rust, and Swift checkers without changing the Checker interface or Options struct?
